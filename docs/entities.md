# Описание сущностей для базы данных 

## Сущность Task

### Поля 

- id 
- status (навзание колонок по сути)
- title (название задачи)
- description (то что будет показываться на dashboard)
- priority 
- deadline (выпольнить до)

### FK и связи 

- tags - связь многие ко многим, через дополнительную таблицу tags 
Фреймворки обчно позволяют такие связи показывать через поля объектов 
Например ```task.tags``` он будет содержать список тэгов для конкретной карточки ["one", "two"] 


## Сущность User

### Поля 

- id
- name UNIQUE(по сути логин но по скольку для логина будет другая таблица здесь будет храниться как имя) 
- email
- is_admin

### Роли 

Обычно роли задаются через фреймворк и все поведение тоже 
В нашем случае 

## Сущность Login

### Пол

- id
- user_name (FK на User.name)
- password (хранит пароль в хэшировнном виде)

## Сущность Tag 

### Поля

 - id
 - tag UNIQUE (уникальный строковый тип, можно сделать ограничение на количестов символов, чтобы не было слишком длинных тэгов)

###  FK

Имеет связь с таблицей Task многие ко многим 
Делается через отдельную таблицу например tags
